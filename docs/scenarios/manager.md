# Сценарий: Менеджер

**Персонаж:** Мария, менеджер по релокации. Ведёт кандидатов от заявки до трудоустройства.
Работает в системе каждый день, одновременно ведёт несколько кейсов.

---

## Утро — Вход в систему

**URL:** `/login`

Мария вводит email и пароль.

**API:** `POST /api/v1/auth/login`

**Редирект:** `/dashboard`

---

## Дашборд менеджера

**URL:** `/dashboard`

**Что видит Мария:**
- Список активных кейсов с текущими этапами
- Кейсы с просроченным дедлайном (выделены красным)
- Кейсы с непрочитанными сообщениями (счётчик в карточке)
- Кейсы ожидающие действия менеджера (без назначенного следующего шага)

**Пример списка:**

| Кандидат | Вакансия | Этап | Дедлайн | Непрочитанных |
|---------|---------|------|--------|---------------|
| Азиз Каримов | Сбор клубники, Бавария | interview | Пятница 14:00 | 2 |
| Нилуфар Исмаилова | Медсестра, Мюнхен | documents | — | 0 |
| Рустам Назаров | Склад, Гамбург | applied | — | 1 |

---

## Новая заявка — Назначение на кейс

**Мария видит новый кейс** (этап `applied`, менеджер не назначен).

**URL:** `/admin/cases`

**API:** `GET /api/v1/cases`

**Действие:** открывает кейс, назначает себя менеджером.

**API:** `PATCH /api/v1/cases/{case_id}`
```json
{ "manager_id": "maria_uuid" }
```

**Мария проверяет:**
1. Профиль кандидата — тип, уровень языка, опыт
2. Загруженные документы — паспорт, CV
3. Сопроводительное письмо из заявки

---

## Статус заявки → reviewing

**API:** `PATCH /api/v1/applications/{id}/status`
```json
{ "status": "reviewing" }
```

**Мария пишет кандидату в чат:**
> «Здравствуйте! Я ваш менеджер. Ваша заявка принята на рассмотрение. Свяжусь с вами насчёт интервью.»

---

## Чеклист перед переходом на следующий этап

**API:** `GET /api/v1/cases/{case_id}/requirements`

Мария видит:
```
Текущий этап: applied
Следующий:    interview

Требования:
✅ Заявка в статусе reviewing
✅ Менеджер назначен

Можно перейти: ДА
```

---

## Переход: applied → interview

**Мария переводит кейс на этап `interview`.**

**API:** `POST /api/v1/cases/{case_id}/advance`

**Устанавливает дедлайн:**

**API:** `PATCH /api/v1/cases/{case_id}`
```json
{ "stage_deadline": "2025-06-06" }
```

**Система автоматически:**
- Записывает в `case_stage_history`: кто перевёл, когда, с какого на какой этап
- Отправляет уведомление кандидату: «Этап изменён: Интервью. Дедлайн: пятница»

**Мария пишет в чате:**
> «Интервью назначено на пятницу, 14:00 по Ташкенту. Видео-звонок в Zoom. Ссылку пришлю в четверг.»

---

## Подготовка к интервью

**Мария готовит:**
- Ссылку на Zoom-встречу с работодателем
- Отправляет её в чате в четверг

**Если кандидат задаёт вопросы в чате** — Мария отвечает через интерфейс менеджера:
> «Одевайтесь официально. Говорите медленно и чётко. Работодатель понимает акцент, главное — уверенность.»

---

## После интервью

**Мария получает обратную связь от работодателя.**

**Проверяет чеклист:**

**API:** `GET /api/v1/cases/{case_id}/requirements`
```
Текущий этап: interview
Следующий:    documents

Требования:
✅ Менеджер добавил заметку (подтверждение интервью)

Можно перейти: ДА
```

**Мария добавляет заметку** через `PATCH /cases/{case_id}`:
```json
{ "notes": "Интервью прошло 06.06.2025. Работодатель подтвердил принятие кандидата." }
```

**Переход:** `interview → documents`

**Мария пишет в чате:**
> «Отличное интервью! Работодатель принял вас. Теперь нужно загрузить диплом — вижу его ещё нет в системе.»

---

## Проверка документов

**URL:** `/admin/cases/{id}/documents`

Мария видит загруженные кандидатом документы.

**Чеклист:**
```
Текущий этап: documents
Следующий:    visa

Требования:
✅ Паспорт
✅ CV
❌ Диплом — не загружен

Можно перейти: НЕТ
```

**Мария ждёт** пока кандидат загрузит диплом. После загрузки система обновляет чеклист.

```
✅ Паспорт
✅ CV
✅ Диплом

Можно перейти: ДА
```

**Переход:** `documents → visa`

---

## Этап: visa

**Система блокирует переход `visa → relocation`** пока не загружена виза.

**Мария видит в чеклисте:**
```
❌ Виза — не загружена
Можно перейти: НЕТ
```

**Мария пишет кандидату:**
> «Все документы приняты! Теперь нужно оформить визу. В разделе "Обучение" есть подробная инструкция. Как только получите — загрузите скан в систему.»

После загрузки визы:
```
✅ Виза
Можно перейти: ДА
```

**Переход:** `visa → relocation`

---

## Этап: relocation

**Мария отправляет финальный пакет информации в чате:**
- Дата начала работы
- Адрес работодателя
- Контакт HR-менеджера в клинике / на ферме
- Инструкция по Anmeldung (регистрация по месту жительства)

**Устанавливает дедлайн** — дата начала работы.

---

## Завершение кейса

**После подтверждения кандидата о прибытии** Мария закрывает кейс:

**API:** `POST /api/v1/cases/{case_id}/advance`
```json
{ "confirm": true }
```

Кейс переходит в `completed`.

---

**API:** `GET /api/v1/cases/{case_id}/history`

Мария видит полную хронологию:

| relocation → completed | Мария Петрова | 05.09.2025 | Кандидат приступил к работе |

## Ведение нескольких кейсов одновременно

**Мария переключается между кейсами:**

- **Кейс Азиза** — этап `interview`, ждёт видео-звонка завтра
- **Кейс Нилуфары** — этап `documents`, ждёт нострификацию
- **Кейс Рустама** — этап `applied`, только назначила себя

**Уведомления в реальном времени через WebSocket:**
- Новое сообщение в чате → красная точка на карточке кейса
- Кандидат загрузил документ → уведомление в шапке
- Дедлайн просрочен → карточка кейса выделена красным

---

## Итог — Что делает менеджер на каждом этапе

| Этап | Действия менеджера |
|------|-------------------|
| `applied` | Назначить себя, проверить профиль и документы, написать в чат |
| `interview` | Согласовать дату, отправить ссылку, поддержать кандидата |
| `documents` | Проверить полноту документов, при необходимости запросить недостающие |
| `visa` | Проконсультировать по визовому процессу, дождаться загрузки визы |
| `relocation` | Отправить финальный пакет, дать контакты работодателя |
| `completed` | Подтвердить завершение, закрыть кейс |
