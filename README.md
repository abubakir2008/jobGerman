## О проекте

Единый сервис, который:

- соединяет кандидатов и работодателей
- разделяет студентов (сезонная работа) и специалистов (полная занятость)
- сопровождает пользователя на каждом этапе — от отклика до трудоустройства

---

## 👥 Роли пользователей

| Роль                    | Описание                                                               |
| ----------------------- | ---------------------------------------------------------------------- |
| **Candidate (Student)** | Соискатель сезонной работы                                             |
| **Candidate (Worker)**  | Соискатель на полную занятость                                         |
| **Manager**             | Ведёт кандидатов по этапам релокации, контролирует документы и статусы |
| **Admin**               | Управляет вакансиями, категориями и обучающим контентом                |

## 🧱 Модули системы

### 1. Auth Service

- Регистрация и логин
- JWT авторизация

### 2. User Profile

- Данные кандидата
- Тип пользователя: `student` / `worker`
- Уровень языка, опыт, готовность к релокации

### 3. Job Module

- Вакансии и категории
- Фильтрация по типу: `seasonal` / `full-time`

### 4. Application System

- Отклики на вакансии
- Статусы заявок

### 5. Relocation Case _(ключевая логика)_

Система сопровождения кандидата по этапам:

```
applied → interview → documents → visa → relocation
```

### 6. Documents Module

- Паспорт, CV, дипломы, переводы

### 7. Media / Learning Module

Обучающий контент по категориям:

| Тип контента                   | Категории                                  |
| ------------------------------ | ------------------------------------------ |
| Видео, изображения, инструкции | Визы, интервью, переезд, работа в Германии |

## 🗃 База данных

**PostgreSQL** — основные таблицы:

| Таблица              | Назначение             |
| -------------------- | ---------------------- |
| `users`              | Аккаунты пользователей |
| `candidate_profiles` | Профили кандидатов     |
| `jobs`               | Вакансии               |
| `job_categories`     | Категории вакансий     |
| `applications`       | Отклики                |
| `relocation_cases`   | Кейсы сопровождения    |
| `documents`          | Метаданные документов  |
| `media`              | Обучающий контент      |

> **Файлы не хранятся в БД.** Используется MinIO или Amazon S3. В БД хранятся только `file_url` и метаданные.

```
GET    /profile

### Jobs

```
GET    /jobs

```

### Applications

```
POST   /applications

```

### Relocation Cases

```

## 🧠 Бизнес-логика

**Разделение кандидатов:**

- `student` → только `seasonal` вакансии
- `worker` → только `full-time` вакансии

**Автоматизация при отклике:**

1. Создаётся `Application`
2. Создаётся `RelocationCase`
3. Назначается менеджер
